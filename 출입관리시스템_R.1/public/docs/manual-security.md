## 1. 시스템 개요

### 1.1 특수경비대 역할
특수경비대는 QR 스캐너를 사용하여 방문자의 출입을 관리합니다. 방문자가 제시하는 QR코드를 스캔하여 입장/퇴장을 처리합니다.

### 1.2 출입 게이트
| 게이트 | 코드 | 설명 |
|--------|------|------|
| **정문** | main | 터미널 정문 출입구 |
| **1부두** | pier_1 | 1부두 출입구 |
| **2부두** | pier_2 | 2부두 출입구 |

### 1.3 처리 흐름
```
방문자 QR 제시 → QR 스캔 → 유효성 확인 → 입장허용/거부 → 자동 기록
```

## 2. QR 스캐너 사용법

### 2.1 스캐너 접속
1. 스마트폰/태블릿 브라우저에서 접속
2. **URL**: https://v0-lng-tml.vercel.app/scanner
3. 로그인 (슈퍼어드민 계정)

### 2.2 스캐너 화면 구성

#### 상단 영역
- **게이트 선택**: 정문 / 1부두 / 2부두
- **방향 선택**: 입장(ENTRY) / 퇴장(EXIT)

#### 카메라 영역
- QR코드 스캔 영역
- 카메라 권한 허용 필요

#### 하단 영역
- 스캔 안내 메시지

### 2.3 게이트 설정
1. 상단에서 **현재 위치에 해당하는 게이트** 선택
2. 선택한 게이트로 출입 기록이 저장됨

### 2.4 방향 설정
| 방향 | 용도 |
|------|------|
| **입장 (ENTRY)** | 방문자가 터미널로 들어올 때 |
| **퇴장 (EXIT)** | 방문자가 터미널에서 나갈 때 |

**중요**: 입장/퇴장 방향을 정확히 설정해야 출입 기록이 올바르게 저장됩니다.

---

## 3. 입장/퇴장 처리

### 3.1 입장 처리 절차
1. 스캐너에서 **"입장(ENTRY)"** 선택
2. 방문자에게 **QR코드 제시** 요청
3. 카메라로 QR코드 스캔
4. **결과 화면 확인**
   - 입장 허용: 녹색 화면 + 체크 아이콘
   - 입장 거부: 빨간색 화면 + X 아이콘
5. 허용 시 입장 안내, 거부 시 사유 확인

### 3.2 퇴장 처리 절차
1. 스캐너에서 **"퇴장(EXIT)"** 선택
2. 방문자에게 **QR코드 제시** 요청
3. 카메라로 QR코드 스캔
4. 결과 화면 확인 후 퇴장 안내

### 3.3 자동 이동
- 스캔 결과 화면은 **3초 후 자동으로** 스캐너 화면으로 복귀
- **"다음 QR 스캔하기"** 버튼으로 즉시 이동 가능

## 4. 결과 화면 해석

### 4.1 입장 허용 (ALLOW)

#### 화면 표시
- **배경색**: 녹색
- **아이콘**: 체크 표시
- **메시지**: "입장 허용" 또는 "퇴장 허용"

#### 표시 정보
| 항목 | 설명 |
|------|------|
| 방문자명 | 출입하는 방문자 이름 |
| 소속 | 방문자 소속 |
| 방문 지역 | 허용된 방문 구역 |
| 방문 기간 | 유효 기간 |

#### 처리
- 입장/퇴장 기록 자동 저장
- 방문자 입장 안내

### 4.2 입장 거부 (DENY)

#### 화면 표시
- **배경색**: 빨간색
- **아이콘**: X 표시
- **메시지**: "입장 거부" + 거부 사유

#### 주요 거부 사유
| 사유 | 설명 | 조치 |
|------|------|------|
| **유효하지 않은 QR코드** | QR코드가 시스템에 없음 | 담당자 확인 요청 |
| **방문 기간 초과** | 방문 기간이 지남 | 재신청 안내 |
| **방문 기간 이전** | 방문 기간이 아직 안 됨 | 방문 예정일 확인 |
| **승인되지 않은 출입증** | 아직 승인 안 됨 | 담당자에게 승인 요청 |
| **허용되지 않은 구역** | 해당 게이트 출입 권한 없음 | 담당자 확인 |

#### 처리
- 입장 불허, 사유 안내
- 필요 시 담당자 연락 안내

## 5. 출입 이력 조회

### 5.1 관리자 페이지 접속
- **URL**: https://v0-lng-tml.vercel.app/admin/qr
- 슈퍼어드민 계정으로 로그인

### 5.2 정문 출입현황
1. **"정문 출입현황"** 탭 선택
2. 날짜 선택 (오늘/특정일/범위)
3. 출입 이력 목록 확인

### 5.3 부두 출입현황
1. **"부두 출입현황"** 탭 선택
2. **1부두** 또는 **2부두** 선택
3. 해당 부두 출입 이력 확인

### 5.4 조회 기능

#### 날짜 네비게이션
- **좌/우 화살표**: 이전/다음 날짜로 이동
- **달력 아이콘**: 특정 날짜 선택

#### 범위 검색
1. **"범위 검색"** 버튼 클릭
2. 시작일/종료일 선택
3. **"검색"** 클릭

#### 새로고침
- **새로고침 버튼** 클릭으로 최신 데이터 조회
- 범위 검색 상태가 초기화되고 선택 날짜 기준으로 조회

### 5.5 이력 정보

#### 인원별 정리
| 항목 | 설명 |
|------|------|
| 방문자명 | 방문자 이름 (클릭 시 첨부파일 확인) |
| 소속 | 방문자 소속 |
| 입장 시간 | 입장 QR 스캔 시간 |
| 퇴장 시간 | 퇴장 QR 스캔 시간 |
| 첨부파일 | 항만이수증 유무 (O/X) |

#### 첨부파일 확인
- 방문자명 클릭 → 첨부파일(항만이수증) 확인/다운로드
- **신청자**와 **동행인**의 첨부파일은 각각 별도 관리

## 6. 문제 해결

### 6.1 QR코드가 스캔되지 않음

#### 원인 및 해결
| 원인 | 해결 방법 |
|------|----------|
| 화면이 어두움 | 방문자에게 화면 밝기 높이기 요청 |
| QR코드 손상 | 새로고침 또는 재접속 요청 |
| 카메라 문제 | 스캐너 페이지 새로고침 |
| 반사광 | 각도 조절 |

### 6.2 카메라가 작동하지 않음

#### 해결 방법
1. 브라우저 카메라 권한 확인
2. 페이지 새로고침
3. 다른 브라우저로 시도
4. 기기 재시작

### 6.3 방문자가 QR코드를 분실함

#### 처리 방법
1. 방문자에게 **접수번호** 확인
2. 담당자에게 연락하여 QR 재발송 요청
3. 또는 출입증 조회 페이지에서 QR 확인

### 6.4 네트워크 오류

#### 해결 방법
1. Wi-Fi 또는 모바일 데이터 연결 확인
2. 페이지 새로고침
3. 잠시 후 재시도

### 6.5 긴급 상황

#### 시스템 장애 시
1. 수기로 방문자 정보 기록 (이름, 소속, 시간)
2. IT 관리팀에 즉시 연락
3. 시스템 복구 후 수기 기록 입력
